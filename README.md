## Инструкци
1. Откройте терминал в папке проекта.
2. Выполните установку зависимостей: `npm install`.
3. Запустите тесты: `npm test`.

### Выявлены критические ошибки
Эндпоинт получения корзины по ID (GET /carts/{cartId}) отдает данные без авторизации.
На реальном коммерческом проекте я бы классифицировал это как уязвимость высокой критичности и завел баг-репорт.
Аналогичные критические уязвимости выявлены на всех других методах API (добавление, обновление, удаление,...)

В нашем случае, будем считать, что это особенность тестового API.

### Выявлены ошибки
Выявлены 3 ошибки:

**При авторизация по токену без Bearer**
То есть в документации написано:
`'Authorization': 'Bearer /* YOUR_ACCESS_TOKEN_HERE */'`
А реально работает:
`'Authorization': '/* YOUR_ACCESS_TOKEN_HERE */'`

**Создание корзины**
Количество товаров не соответствует отправленному. Точнее, данные перепутаны.

**Создание корзины с неверными данными в Body (не задан ID продукта)**
При добавлении товара без ID возвращается не ожидаемый код 400, а код 201 Created.

**Обновление корзины без новых продуктов**
При обновлении корзины без передачи новых продуктов возвращается не ожидаемый код 400, а код 200 OK.

Аналогичных мелких багов много, то есть практически любые данные на вход дают не вызывают 400, а выдают позитивный код ответа.

Но, как я понимаю, в тестовом задании не нужно ловить все варианты подобных мелких багов.
